## Summary

This PR improves CLI documentation to better support agent and tool harness integrations.

## Changes

### skills.md

- Complete command reference with all flags and examples
- Detailed Playwright usage guide with TypeScript example
- Quick workflow section
- **Important warning about `--attach`**: clarifies that tool call harnesses without automatic process detach should not use this flag, as the verification code will expire when the parent process exits
- Status semantics documentation

### Web/public/llms.txt

- New "Skill File" section explaining how to install and use the skill
- Process model documentation (detached vs foreground)
- Expanded command list with descriptions
- **Warning about `--attach`** for agent harnesses
- Playwright usage example

### CommandLineTool/AGENTS.md

- New "CLI Behavior" section covering:
  - Full command surface
  - Request lifecycle
  - Status semantics (waiting, receiving, ready, expired, orphaned, error, missing)
  - Session commands behavior
  - Config keys
- "Daemon Process Model" section explaining detached execution
- Exit codes documentation
- Suggested automation patterns

## Why

The existing documentation was minimal and did not clearly explain:

1. The detached process model which is critical for long-running operations
2. The implications of using `--attach` in different environments
3. How to integrate with automation tools like Playwright

This documentation helps LLM agents and CI/CD systems use Cookey correctly without falling into common traps like using `--attach` without proper process management.
